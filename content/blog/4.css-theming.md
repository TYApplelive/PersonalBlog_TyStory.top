# CSS 变量与站点主题的收口方式

这个博客的风格不是默认白底卡片，而是偏复古、偏电影感的视觉方向。只要项目开始往这种明确风格推进，就不能再把颜色和间距零散写在每个组件里。

## 为什么要先做主题层

- 保证浅色纸面和深色背景之间的对比关系稳定
- 控制按钮、卡片、标签这些元素的风格一致性
- 降低后续改版时的联动成本

## 变量收口之后的好处

当所有页面都基于一套主题变量工作时，首页、关于页、博客列表页和详情页虽然用途不同，但仍然会保持统一气质，不会每个页面像不同项目拼起来的一样。

> 设计系统不是为了限制页面，而是为了让变化有边界。

对这个站点来说，CSS 变量的意义不只是“好改色”，而是让复古电影感这条视觉线能够持续成立。
