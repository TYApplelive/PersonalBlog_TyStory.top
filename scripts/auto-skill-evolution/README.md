# Auto-Skill Evolution - 自动技能进化系统

## 概述

Auto-Skill Evolution 是一个自动化机制，用于在 Nuxt 项目中管理 `.lingma/skills` 和 `.trae/skills` 目录。当用户确认任务完成时，该机制会自动提取知识、检查重复、更新或创建技能文件。

## 目录结构

```
scripts/auto-skill-evolution/
├── cli.js           # 命令行入口
├── engine.js        # 核心引擎
├── config.js        # 配置文件
├── extractor.js     # 知识提取器
├── deduplicator.js  # 去重检查器
├── validator.js     # 验证器
└── generator.js     # 技能生成器
```

## 工作流程

```
用户确认任务完成
        ↓
触发技能进化机制
        ↓
1. 知识提取 (extractor.js)
   - 分析任务描述
   - 提取代码模式
   - 识别最佳实践
        ↓
2. 扫描现有技能
   - 扫描 .lingma/skills
   - 扫描 .trae/skills
        ↓
3. 去重检查 (deduplicator.js)
   - 计算相似度
   - 判断是否重复
        ↓
4. 生成/更新技能 (generator.js)
   - 创建新技能 OR
   - 更新现有技能
        ↓
5. 验证 (validator.js)
   - 检查格式
   - 检查内容
   - 输出验证结果
```

## 使用方法

### 方法 1: 命令行直接指定参数

```bash
node scripts/auto-skill-evolution/cli.js \
  --task "修复路由跳转问题" \
  --solution "将 blog.vue 改为 blog/index.vue" \
  --files "app/pages/blog/index.vue,app/pages/blog/[id].vue"
```

### 方法 2: 从 JSON 文件读取上下文

```bash
# 1. 创建上下文文件
cat > context.json << 'EOF'
{
  "taskDescription": "修复博客列表页导航跳转 Bug",
  "solution": "将 blog.vue 重命名为 blog/index.vue 解决路由冲突",
  "modifiedFiles": [
    "app/pages/blog/index.vue",
    "app/pages/blog/[id].vue"
  ],
  "codeChanges": {
    "added": ["app/pages/blog/index.vue"],
    "removed": ["app/pages/blog.vue"],
    "modified": ["app/pages/blog/[id].vue"]
  }
}
EOF

# 2. 执行技能进化
node scripts/auto-skill-evolution/cli.js --context context.json
```

### 方法 3: 在代码中调用

```javascript
import { AutoSkillEvolutionEngine } from "./scripts/auto-skill-evolution/engine.js";

const engine = new AutoSkillEvolutionEngine();

const result = await engine.evolve({
  taskDescription: "修复路由跳转问题",
  solution: "将 blog.vue 改为 blog/index.vue",
  modifiedFiles: ["app/pages/blog/index.vue"],
  codeChanges: {
    added: ["app/pages/blog/index.vue"],
    removed: ["app/pages/blog.vue"],
  },
});

console.log(result);
```

## 配置说明

配置文件位于 `scripts/auto-skill-evolution/config.js`

### 主要配置项

```javascript
export const AutoSkillEvolutionConfig = {
  // 技能目录路径
  skillDirectories: {
    lingma: ".lingma/skills",
    trae: ".trae/skills",
  },

  // 去重配置
  deduplication: {
    similarityThreshold: 0.75,  // 相似度阈值
    checkFields: ["title", "description", "keywords", "content"],
  },

  // 验证配置
  validation: {
    requiredFields: ["name", "description", "content"],
    encoding: "utf-8",
    checkMarkdownFormat: true,
  },
};
```

## 去重机制

### 相似度计算

系统使用多维度相似度计算：

1. **标题相似度**: Jaccard 相似度算法
2. **描述相似度**: 文本内容匹配
3. **关键词匹配**: 关键词在内容中的出现率
4. **分类匹配**: 问题分类与技能内容的匹配度

### 去重策略

- 相似度 >= 75%: 认定为重复，更新现有技能
- 相似度 < 75%: 认定为新技能，创建新文件

## 验证规则

### 格式验证

- ✅ 必须有 Markdown 标题
- ✅ 标题层级不能跳跃
- ✅ 代码块必须有语言标识
- ✅ 代码块必须正确闭合

### 内容验证

- ✅ 必须包含必填字段（name, description, content）
- ✅ 内容长度 5-500 行
- ✅ 无未闭合的 HTML 标签

## 集成到工作流

### 在 Git Hook 中使用

```bash
# .git/hooks/post-commit
#!/bin/bash
node scripts/auto-skill-evolution/cli.js \
  --task "$(git log -1 --pretty=%B)" \
  --files "$(git diff --name-only HEAD~1)"
```

### 在 CI/CD 中使用

```yaml
# .github/workflows/skill-evolution.yml
name: Skill Evolution
on:
  push:
    branches: [main]

jobs:
  evolve:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - run: node scripts/auto-skill-evolution/cli.js --context context.json
      - run: git add .lingma/skills .trae/skills
      - run: git commit -m "chore: auto-skill-evolution"
```

## 常见问题

### Q: 如何调整相似度阈值？

A: 编辑 `config.js` 中的 `deduplication.similarityThreshold` 值。

### Q: 如何禁用自动进化？

A: 删除或重命名 `scripts/auto-skill-evolution` 目录。

### Q: 技能文件可以手动编辑吗？

A: 可以。手动编辑后，`_meta.json` 中的 `updatedAt` 时间戳会在下次进化时更新。

### Q: 支持哪些技能分类？

A: 支持以下分类：
- routing: 路由规范
- stateManagement: 状态管理
- dataFetching: 数据获取
- components: 组件开发
- styling: 样式规范
- deployment: 部署配置
- debugging: 调试技巧
- performance: 性能优化
- security: 安全实践
- workflow: 工作流
